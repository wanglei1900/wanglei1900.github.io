SQL 索引查询优化 ​

1 INDEX 索引的概念 ​

在不读取整个表的情况下，索引使数据库应用程序可以更快地查找数据。

更新一个包含索引的表需要比更新一个没有索引的表更多的时间，这是由于索引本身也需要更新。因此，理想的做法是仅仅在常常被搜索的列（以及表）上面创建索引。

  • INDEX： 索引 允许有重复值
  • UNIQUE INDEX： 创建唯一索引
  • DESC ASC： 创建 降序/升序 索引值
  • 复合索引：多个索引用逗号隔开

2 索引的原则 ​

理解这些原则有助于设计高效的索引：

  1. 最左前缀：索引列顺序很重要，等值列在前，范围列在后。

  2. 等值乱序：优化器会自动调整等值条件的顺序以匹配索引。

  3. 高区分度：选择不同值多的列建索引。

  4. 列不计算：避免在索引列上使用函数或表达式。

  5. 扩展索引：通过增加列来扩展已有索引，避免冗余索引。

2.1 最左前缀 ​

核心：索引使用从左到右的顺序，遇到范围查询（>、<、between、like）即停止匹配

范围查询条件在联合索引处应该放在后面 a = 1 and b = 2 and c > 3 and d = 4 如果建立(a,b,c,d)顺序的索引，d是用不到索引的，如果建立(a,b,d,c)的索引则都可以用到，a,b,d的顺序可以任意调整

-- 表结构
CREATE TABLE orders (
  id INT PRIMARY KEY,
  user_id INT,
  status TINYINT,
  amount DECIMAL(10,2),
  order_date DATE
);

-- 索引1: (status, amount, user_id, order_date)
-- 索引2: (status, user_id, order_date, amount)

2.3 高区分度 ​

核心：区分度 = COUNT(DISTINCT col) / COUNT(*)

sql
-- 用户表 100万行
SELECT 
  COUNT(DISTINCT mobile)/COUNT(*) AS mobile_distinct,  -- 0.9999（几乎唯一）
  COUNT(DISTINCT gender)/COUNT(*) AS gender_distinct   -- 0.0005（只有男/女）
FROM users;

优化建议：

搜索手机号：适合单独索引（区分度0.9999）

搜索性别：不适合单独索引（区分度0.0005），需组合其他列

组合索引示例：(company_id, department)（公司内部门区分度可达0.3）

2.4 列不计算 ​

核心：保持列"干净"，避免函数包裹 例子：

sql
-- 错误做法（索引失效）：
SELECT * FROM orders 
WHERE DATE_FORMAT(order_date, '%Y-%m') = '2023-06'

-- 正确做法（使用索引）：
SELECT * FROM orders 
WHERE order_date BETWEEN '2023-06-01' AND '2023-06-30'

2.5 扩展索引 ​

核心：避免冗余索引，优先扩展现有索引

例子：

sql
-- 已有索引
ALTER TABLE orders ADD INDEX idx_status (status);

-- 新需求：常查询 status + user_id
-- ❌ 错误做法：新建独立索引
ALTER TABLE orders ADD INDEX idx_user (user_id);

-- ✅ 正确做法：扩展原索引
DROP INDEX idx_status ON orders;
ALTER TABLE orders ADD INDEX idx_status_user (status, user_id);

3 综合应用案例 ​

场景：电商订单查询

sql
-- 高频查询：
SELECT * FROM orders
WHERE user_id = 1005
  AND status IN (2,3) 
  AND amount BETWEEN 100 AND 500
  AND YEAR(order_date) = 2023;  -- ❌ 问题点
1
2
3
4
5
6

优化步骤：

改写计算：YEAR(order_date) = 2023 → order_date BETWEEN '2023-01-01' AND '2023-12-31'

按最左前缀设计索引：(user_id, status, order_date, amount)

user_id(等值) → status(IN) → order_date(范围) → amount(范围)

调整列顺序：(user_id, order_date, status, amount)

理由：order_date 范围查询后，status 和 amount 无法用索引

最终方案：

sql
ALTER TABLE orders 
ADD INDEX idx_user_date_status (user_id, order_date, status, amount);

-- 优化后查询：
SELECT * FROM orders
WHERE user_id = 1005
  AND order_date BETWEEN '2023-01-01' AND '2023-12-31'
  AND status IN (2,3) 
  AND amount BETWEEN 100 AND 500;
